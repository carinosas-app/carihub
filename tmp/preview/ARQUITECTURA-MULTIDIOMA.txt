ARQUITECTURA MULTIDIOMA — traducción automática (fases futuras)
===============================================================

ESTADO ACTUAL
- Solo documentación de arquitectura
- Sin i18n, sin APIs de traducción, sin selector de idioma en mockups
- public/index.html y mockups actuales SIN modificar por esta función

OBJETIVO
Que personas de distintos países puedan escribir o dictar en su idioma y que
el sistema traduzca automáticamente para que visitantes, perfiles, anunciantes
y admin se entiendan entre sí.

ALCANCE PREVISTO (no implementar aún)
- Mensajes internos entre usuarios y perfiles
- Soporte y panel admin
- Registro de perfiles
- Registro de banners / anunciantes
- Asistentes IA (ver ARQUITECTURA-AGENTES.txt)
- Descripciones públicas de perfiles y anuncios
- Denuncias y mensajes de moderación

ACTORES
- visitante   → lee en su idioma preferido
- perfil      → publica y chatea en su idioma
- anunciante  → gestiona campañas en su idioma
- admin       → revisa contenido con traducción asistida
- moderador   → denuncias y mensajes en idioma original + traducción

PRINCIPIOS DE DISEÑO (para no cerrar el HTML)
----------------------------------------------
1. Textos de UI en HTML real, nunca dentro de imágenes (SEO + traducción).
2. Separar:
   - texto de interfaz (labels, botones, ayudas)
   - contenido generado por usuario (bio, mensajes, denuncias)
3. Guardar siempre el idioma detectado o elegido del emisor.
4. Guardar traducción como capa derivada, no reemplazar el original.
5. Mostrar al receptor la versión en su idioma; opción “ver original”.
6. Entrada por texto y por voz (dictado) en fase posterior.

CONVENCIONES HTML/CSS PREPARADAS (fase futura)
----------------------------------------------
Atributos sugeridos — no activar en mockup actual:

  <html lang="es" data-ui-locale="es">
  <body data-content-locale="es">

  <p data-i18n-key="home.hero.title">...</p>
  <textarea data-user-content data-lang="es" data-translate-on-send="true">

  <article data-public-profile data-lang="es" data-show-translation="auto">

Flujo de mensajería (conceptual)
--------------------------------
1. Usuario escribe/dicta en idioma A
2. Sistema detecta o confirma idioma A
3. Al enviar: guarda mensaje original + metadatos (lang, timestamp)
4. Traduce a idioma B del destinatario (o a idiomas de participantes)
5. UI muestra traducción; moderación/admin puede ver ambos

INTEGRACIÓN CON OTROS SISTEMAS FUTUROS
- Temas visuales (ARQUITECTURA-TEMAS.txt): independiente del idioma
- Agentes IA (ARQUITECTURA-AGENTES.txt): responder en idioma del contexto
- Moderador IA: revisar original + traducción antes de publicar si aplica
- Editor visual perfil/dashboard: textos traducibles como capa, no en canvas

PÁGINAS / MÓDULOS AFECTADOS (cuando se implemente)
- Home y búsqueda
- Perfil público
- Mensajería interna
- Registro perfil / registro banner
- Dashboard perfil / dashboard anunciante
- Admin / moderación / denuncias
- Asistentes IA por contexto

NO IMPLEMENTADO AÚN (por diseño)
- Selector de idioma en UI
- localStorage / perfil de preferencia de idioma
- API de traducción (Google, DeepL, modelo propio, etc.)
- Dictado por voz
- Traducción automática en tiempo real del chat
- Publicación multilingüe indexable (hreflang, sitemaps por idioma)

REGLAS PARA MOCKUPS Y HOME HTML
- No añadir librerías i18n al mockup actual
- No duplicar páginas por idioma en esta fase
- Mantener textos en español en mockup; estructura HTML lista para i18n después
